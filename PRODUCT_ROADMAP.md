# 火山AI漫剧 产品路线图 v1

## 产品定位

火山AI漫剧不只是一个无限画布节点工具，而是一个面向漫剧、短剧、AI 分镜、AI 图像、AI 视频生产的桌面级创作工作台。

目标形态：

- 前期：剧本、资产、分镜、镜头提示词生产
- 中期：图片、视频、音频、字幕批量生成
- 后期：时间线剪辑、成片导出、项目归档

一句话目标：

做成一个真正能把“小说/创意”转成“可批量生产的 AI 影视项目”的成熟软件。

---

## 当前版本判断

当前项目已经具备这些基础：

- 无限画布节点系统
- 小说转剧本工厂
- 全局 API 管理
- Gemini 网页生图
- 资产库
- 镜头表
- 基础队列
- ComfyUI 工作流接入

但还缺少成熟软件最关键的三层能力：

1. 项目级生产管理
2. 时间线级成片能力
3. 一致性与自动化检查

---

## 一期：必做

这些功能决定软件能不能从“工具集合”升级成“生产系统”。

### 1. 项目资源中心

目标：
统一管理项目里的图片、视频、音频、字幕、脚本、模板、参考图。

功能：

- 资源面板
- 素材导入、重命名、删除、引用位置追踪
- 角色参考图、场景参考图、风格参考图统一归档
- 资源按集 / 场 / 镜头分类
- 缺失资源自动标红

价值：
后续所有节点都不再各自存素材，而是引用项目资源中心。

### 2. 资产一致性系统

目标：
让角色、场景、道具在整个项目里稳定，不漂移。

功能：

- 角色资产卡：外观、服装、颜色、身份、参考图、锁定词
- 场景资产卡：地点结构、光线、时间、材质、视觉标识物
- 道具资产卡：名称、材质、结构、阶段变化
- 镜头直接引用资产 token
- 资产更新后提示受影响镜头

价值：
这是做 AI 漫剧最核心的稳定层。

### 3. 镜头表升级

目标：
让镜头表真正变成生产调度表，而不是静态表格。

功能：

- 镜头状态：待写、待生图、待生视频、待配音、完成
- 每镜头支持：
  - 图像提示词
  - 视频提示词
  - 负面词
  - 参考图
  - 时长
  - 镜头运动
  - 配音文本
  - 音效备注
- 镜头批量入队
- 镜头批量改参数
- 镜头批量重试失败任务

价值：
镜头表会成为整套系统的生产中枢。

### 4. 统一任务队列中心

目标：
把图片、视频、配音、超分、补帧等任务统一管理。

功能：

- 队列优先级
- 等待 / 运行 / 成功 / 失败 / 已取消
- 失败原因展示
- 批量重试
- 批量取消
- 任务结果自动回写到镜头或资产

价值：
成熟软件一定要有统一调度中心，不然任务一多就乱。

### 5. 时间线 v1

目标：
从“生产素材”走向“拼接成片”。

功能：

- 镜头拖入时间线
- 调整顺序
- 调整时长
- 添加转场
- 添加配音轨
- 添加字幕轨
- 添加音乐轨
- 预览导出

价值：
这是软件从创作工具走向成片工具的分水岭。

## 二期：应做

这些功能会显著拉开和普通 AI 工具的差距。

### 1. 智能检查系统

目标：
自动发现项目缺项和逻辑冲突。

功能：

- 缺失角色资产
- 缺失场景资产
- 镜头未绑定角色 / 场景 / 道具
- 提示词过短
- 剧本和镜头不一致
- 角色命名不统一
- 连续镜头状态冲突

价值：
等于给软件加了一个项目监制。

### 2. Prompt 工厂

目标：
统一生成不同模型所需的提示词版本。

功能：

- 图像 Prompt 模板库
- 视频 Prompt 模板库
- 风格预设
- 模型预设
- 资产卡自动转镜头 Prompt
- Prompt 历史记录

价值：
减少重复劳动，也方便不同 API 之间切换。

### 3. 音频系统

目标：
把图像视频生产补到真正可交付。

功能：

- 角色配音
- 背景音乐
- 音效库
- 台词自动配音
- 音频批量挂到镜头
- 字幕自动生成与对轴

### 4. 模板系统

目标：
把“经验”沉淀成模板。

功能：

- 剧本模板
- 审稿模板
- 资产模板
- 分镜模板
- Prompt 模板
- API 预设模板
- 模板导入导出

### 5. 性能模式

目标：
保证大项目仍然可用。

功能：

- 节点过多时自动降级阴影
- 连线简化模式
- 小地图
- 分页加载或视口虚拟化
- 项目分集加载

## 三期：可做

这些功能会把软件进一步拉高到“专业生产台”。

### 1. 智能导演助手

功能：

- 根据剧本自动给出镜头拆分建议
- 根据镜头自动补出图像 Prompt
- 自动给出镜头运动建议
- 自动标出节奏弱点

### 2. 多风格生产

功能：

- 真人写实
- CG 电影感
- 动漫 / 国漫
- 统一风格锁定面板

### 3. 协作能力

功能：

- 工程共享
- 节点评注
- 审稿反馈
- 分工生产

### 4. 发布导出

功能：

- 短视频导出
- 横竖版适配
- 字幕烧录
- 封面图导出
- 项目归档包

---

## 界面结构建议

建议长期稳定成 5 个一级区域：

1. 画布区  
   负责流程搭建和节点组织

2. 项目区  
   负责剧集、场次、资源、模板、历史版本

3. 资产区  
   负责人设、场景、道具、参考图

4. 生产区  
   负责镜头、队列、API、生成参数

5. 时间线区  
   负责成片编辑、音频、字幕、导出

## 最短结论

接下来不要再把重点放在“再加一个单点节点功能”。

真正能让软件成熟的，是把下面这条链打通：

剧本 -> 资产 -> 分镜 -> 镜头 -> 图片 / 视频 / 音频 -> 时间线 -> 导出

只要这条生产链跑顺，软件就会从“能玩”变成“能做项目”。
